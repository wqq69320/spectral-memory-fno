# M18 Decision Log

This log records implementation decisions for the presentation artifact pack.
It does not add benchmark claims.

## Decisions Made

- Created `scripts/create_m18_presentation_artifacts.py` as the single
  reproducible M18 entrypoint.
- Used the M17 full-test rollout artifacts as source data instead of rerunning
  training inside the presentation script.
- Prioritized the dynamic Navier-Stokes2D fixture, FNO2D, last-frame
  persistence, and optimized SM-FNO2D v3.
- Used dark-theme-compatible Matplotlib styling for the final plots, tables,
  summary card, and GIFs.
- Selected the visualized held-out sample automatically by largest step-36
  persistence error, unless `--sample-index` is provided.
- Generated static 2D panels, 3D-style surface triptychs, comparison plots,
  Markdown/CSV/PNG tables, a summary card, and GIFs.
- Extended the M18 artifact generator with dark-theme 3D-style animated
  rollouts for the true target, persistence, FNO2D, optimized SM-FNO2D v3, and
  a side-by-side comparison.
- Stored animated 3D-style outputs under
  `results/figures/m18_presentation/3d_style/`.
- Added the dynamic Transformer2D diagnostic config and incorporated its
  full-test rollout artifact into the final M18 comparison table, plots, GIFs,
  and step-36 panel.
- Labeled Transformer2D as a temporal-attention baseline applied independently
  per spatial grid cell, not a full spatiotemporal Transformer.

## Alternatives Considered

- Deferring Transformer2D from the first M18 pack vs adding a same-protocol
  dynamic fixture artifact. The follow-up adds Transformer2D after training and
  evaluating it under the same dynamic fixture split and 36-step rollout
  protocol.
- Creating a slide deck directly. M18 instead creates reusable figures, GIFs,
  and index files so slides can be assembled without locking into a deck tool.
- Exporting MP4s by default. GIFs are generated by default; MP4 export is
  optional and depends on an available ffmpeg writer.
- Encoding 3D-style animations as true 3D simulations. M18 explicitly treats
  them as surface renderings of 2D scalar vorticity fields.
- Treating Transformer2D as a full attention baseline. The existing
  implementation attends over temporal tokens at each spatial grid point, which
  is the honest comparison supported by this codebase.

## Implementation Rationale

The final talk needs clear visual evidence for the research arc: rollout
stability, persistence as a necessary baseline, dynamic-skill diagnostics, and
the optimized SM-FNO2D v3 result. The M18 script recomputes metrics from the
stored full-test rollouts so the figures and tables remain traceable.

The artifact index is generated under `results/tables/m18_presentation/` to make
slide selection fast while keeping generated outputs out of git.

The 3D-style animations use fixed color scale, fixed z-axis limits, and a fixed
camera view across frames to avoid flicker during slide playback.

The attention-baseline follow-up uses the same dynamic fixture split, seed, and
36-step rollout horizon as FNO2D and optimized SM-FNO2D v3. This keeps the
presentation comparison aligned without turning it into a benchmark claim.

## Known Limitations

- The presentation pack is based on one synthetic dynamic fixture and one
  optimized diagnostic run.
- Positive dynamic skill remains a fixture-specific diagnostic result, not a
  general benchmark conclusion.
- MP4 output is deferred unless an ffmpeg writer is installed.
- The automatic sample selection favors visual dynamism, not statistical
  representativeness.
- The 3D-style outputs are visualizations of 2D vorticity rollouts, not true 3D
  Navier-Stokes predictions.
- Transformer2D is CPU-feasible and intentionally small; it should not be
  interpreted as a tuned or full spatiotemporal attention architecture.
